Lost in Code

  • #group_by Explained!
  • November 1 2014



Once you begin the journey of Enumerables, you are bound to wind up coming across #group_by. While the ruby-doc explanation is somewhat clear to the purpose of calling #group_by, it’s uses are not as easily explained.

The #group_by method will “group” a collection by the block of code you pass it through and return a hash as the result. The keys in this hash will be the result of the code and the values will be the arrays of elements that correspond to the key.

For example:

names = ["James", "Bob", "Joe", "Mark", "Jim"] names.group_by{ |name| name.length}

Will return



{5=>["James"], 3=>["Bob", "Joe", "Jim"], 4=>[“Mark"]}

As you can see from this example, the code requests the length of each item and then assigns a key for each length and value of each element that matches that key. So the ley of “5” is assigned to “James” because it was “grouped” by the length of 5 and then assigned to that array.

The #group_by method is especially useful when you need to separate information that will need to be altered later in your project. By sorting the information and assigning it a key from your parameters, you can now access this information based on your needs.
