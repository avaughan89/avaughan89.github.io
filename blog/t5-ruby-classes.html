  • Keep It Class(y)
  • November 7 2014



Anyone beginning their studies in Ruby will soon come across the word “class.”

Different tutorials and intro to Ruby books will attempt to simplify the idea of "class" but I found that one of the best tutorials came from the website Ruby Classes where they compare the idea of classes and objects to that of a vehicle and its components.  The website offers the simplistic model of Class as:

Class Vehicle {
Number no_of_wheels
Number horsepower
Characters type_of_tank
Number Capacity
Function speeding {
}
Function driving {
} Function halting {
}
}



Which serves to illustrate how “Class” interacts with it’s “Characteristics”. When attempting to explain the concept to a person new to programming, I believe an effective tactic is to relate the item an everyday item. In this case, our “Class” is vehicles and the objects and methods within that class are the characteristics and details about that vehicle. No matter how the objects within the class are being inspected, we can determine that we are still describing that same vehicle.

A “Class” is a set of rules for an object and sets down the groundwork for how the object will behave. When dealing with Object Orientated Programming, defining how objects will interact is crucial.
